## optimizerDuck の機能

Windows 自体は問題ありません。しかし、クリーンインストールには、おそらく聞いたことのないサービス、テレメトリ、プリインストールアプリ、スケジュールタスクが含まれており、それらはすべてバックグラウンドで静かに動作し、CPU、RAM、ディスクを消費しています。同時に、ハードウェアを最大限に活用するための機能の一部はデフォルトで有効になっていません。

optimizerDuck は、ブロートウェアをクリーンアップし、有用な機能を解放するための単一のインターフェースを提供します。

システムのオーバーヘッドを削減し、不要な動作をブロックするためのターゲット設定された調整を適用するとともに、何が実行されているかを確認したり、不要なものを削除したり、問題が発生した場合に変更を元に戻したりできる管理ツールもバンドルしています。

> [!NOTE]
> すべての最適化は手動で適用できます。optimizerDuck はそれらをより簡単に適用できるようにするだけです。

### システム最適化

6 カテゴリにわたる 30 以上の調整。それぞれに明確な説明とリスク評価が付いており、適用前に各変更の内容を正確に把握できます。

| カテゴリ                     | 内容                                                                                                                                      |
| :--------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------- |
| **パフォーマンス**           | RAM に基づく Service Host チューニング、プロセス優先順位調整、キーボード遅延低減、ゲーム向けマルチメディアスケジューラ調整                |
| **プライバシー**             | Windows テレメトリ、エラー報告、広告 ID、位置情報追跡、Cortana、Copilot、コンテンツ配信の提案を無効化                                     |
| **GPU**                      | AMD、NVIDIA、Intel GPU のベンダー固有レジストリ調整（電源状態、クロックゲーティング、表示遅延）                                           |
| **電源**                     | 休止状態と高速スタートアップの無効化、USB セレクティブサスペンドの解除、カスタム高性能電源プランのインストール、Power Throttling の無効化 |
| **ブロートウェアとサービス** | OEM アプリの再インストール動作をブロックし、200 以上の Windows サービスのスタートアップ種類を調整                                         |
| **ユーザーエクスペリエンス** | メニュー表示遅延の削除、タスクバーアニメーションや透明度などの視覚効果を無効化                                                            |

> [!NOTE]
> ここでの最適化は、ユーザーベースの多い有名ツールから調査されたものであり、AI 生成や盲目的に追加されたものはありません。すべての調整は実際の影響に基づいて選択されています。

### カスタマイズ

レジストリを直接操作する必要はなく、トグル、ドロップダウン、数値入力が一箇所にまとめられています。4 つのカテゴリに分かれています：

- **デスクトップ**：アイコンの表示/非表示（PC、ごみ箱、ネットワーク、ユーザーファイル、コントロールパネル）、ショートカット矢印の削除
- **環境設定**：タスクバーの配置、ウィジェット、タスクビューとタスク終了ボタン、秒表示、ダークモード、ファイル拡張子、隠しファイル、クリップボード履歴、コンパクトビュー、スナップアシスト、チェックボックス、クラシックコンテキストメニュー、Bing 検索
- **ゲーム**：ゲームモード、ゲームバー、バックグラウンド録画、マウス加速、全画面最適化、ハードウェアアクセラレーション GPU スケジューリング
- **システム**：起動時の NumLock 有効化

### 内蔵ツール

| ツール                         | 機能                                                                                                                                                     |
| :----------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **システムダッシュボード**     | CPU、RAM、GPU、ストレージドライブ、OS の詳細を一つのパネルで表示                                                                                         |
| **スタートアップマネージャー** | 起動時に実行されるアプリやタスクを確認、オン/オフの切り替え、ファイルの場所を開く                                                                        |
| **スケジュールタスク**         | Windows スケジュールタスクの参照、実行、停止、有効化、無効化、削除                                                                                       |
| **ディスククリーンアップ**     | 一時ファイル、システムキャッシュ、Windows Update の残骸、プリフェッチ、サムネイル、ごみ箱、クラッシュダンプ、古い Windows インストールをスキャンして削除 |
| **ブロートウェア削除**         | 削除可能な AppX パッケージをリスクバッジ（安全、注意、不明）付きで一覧表示                                                                               |

## 安全性

システム設定の変更にはリスクが伴います。optimizerDuck は、元に戻しやすさとユーザーコントロールを基本に設計されています。

詳細なデータプラクティスについては、[プライバシーポリシー](./PRIVACY.md) をご覧ください。

- **自動バックアップ**：すべての変更は元に戻すファイルをローカルフォルダに書き込みます。個別の調整を元に戻したり、すべてをロールバックしたりできます
- **ワンクリックで元に戻す**：UI からワンクリックで適用済みの最適化を元に戻せます
- **リスク評価**：各調整は、潜在的な影響に基づいて「安全」「注意」「上級者向け」に分類されています
- **デフォルトでは適用されない**：ユーザーが選択するまで何も実行されません。ツールが自動的に有効化することはありません
- **復元ポイントの通知**：最初の最適化の前に、Windows の復元ポイントを作成するよう促します

## よくある質問

> [!TIP]
> その他の質問と回答については、[ウェブサイトのFAQ](https://optimizerduck.vercel.app/docs/faq/general)をご覧ください。

### optimizerDuck は安全ですか？

はい。optimizerDuck は完全に**オープンソース**（GPL v3）であり、誰でもソースコードを検査、監査、ビルドできます。すべてのリリースは **GitHub Actions** が公開ソースから自動ビルドしており、隠された変更やビルド後に注入された未署名のバイナリはありません。ご希望であれば、リポジトリをクローンして、`dotnet build` 一発で自分で `.exe` をビルドすることもできます。

アプリはテレメトリ、使用データ、個人情報を一切収集**しません**。[プライバシーポリシー](./PRIVACY.md) をご覧ください。

### optimizerDuck は実際にパフォーマンスを向上させ、レイテンシを低減し、ネットワークを高速化しますか？

役立つ可能性があります。optimizerDuck のすべての最適化は**有名ツール、コミュニティガイド、ハードウェアベンダーの推奨事項から調査されたもの**であり、AI 生成や盲目的に追加されたもの、でっち上げは一切ありません。各調整は、Windows がデフォルトで控えめに設定している実際の設定（Service Host グループ化、GPU 電源状態、ネットワークスロットル、プロセススケジューリングなど）に対処します。

ここに偽のレジストリハックはありません。すべての変更には文書化された目的があり、コミュニティテストとベンダーのドキュメントに裏付けられた実際の効果があります。

### Windows SmartScreen / Defender がダウンロードを警告するのはなぜですか？

optimizerDuck はコード署名されていません。コード署名証明書はオープンソースプロジェクトにとって高額だからです。Windows がインターネットからダウンロードされた未署名の実行ファイルを検出すると、SmartScreen はデフォルトで警告を表示します。これは正常であり、ファイルが安全でないという意味では**ありません**。

回避するには、**「詳細情報」>「実行」** をクリックしてください。それでも心配な場合：

- [ソース](https://github.com/itsfatduck/optimizerDuck) から自分で `.exe` をビルドする
- ANY.RUN などのオンラインサンドボックスにバイナリを提出して検証する

### 問題が発生した場合、変更を元に戻せますか？

はい。すべての最適化は、適用前に元に戻すファイルを作成します。UI からワンクリックで個別の調整を元に戻したり、すべてをロールバックしたりできます。また、最初の最適化の前に Windows のシステム復元ポイントを作成することも促されます。

### Windows 10 と Windows 11 で動作しますか？

はい。optimizerDuck は **Windows 10 (x64)** と **Windows 11 (x64)** をサポートしています。

### 管理者権限は必要ですか？

はい。システム設定と Windows レジストリを変更するため、管理者権限での実行が必要です。

### optimizerDuck は私のデータを収集しますか？

いいえ。アプリにはテレメトリ、分析、外部通信機能は一切含まれていません。完全にオフラインで動作し、データをどこにも送信しません。

### 電源プラン適用後、タスクマネージャーが CPU 100% と表示するのはなぜですか？([#29](https://github.com/itsfatduck/optimizerDuck/issues/29))

デフォルト以外の電源プランで発生するタスクマネージャーの既知の表示バグ，一部のシステムで実際の負荷は正常にもかかわらず CPU 100% と誤表示します。視覚的な問題のみで、実際のパフォーマンスには**影響せず**、過熱の原因にもなりません。不要であれば、この最適化をオフにしてください。

## トラブルシューティング

> [!TIP]
> 詳細なガイダンスと既知の問題の回避策については、[トラブルシューティングページ](https://optimizerduck.vercel.app/docs/faq/troubleshooting)をご覧ください。

### アプリが起動しない、または起動時にクラッシュする

**管理者として実行**していることを確認してください。optimizerDuck は昇格された権限を必要とします。それでもクラッシュする場合は、[Releases](https://github.com/itsfatduck/optimizerDuck/releases/latest) から最新バージョンをダウンロードしてください。古いビルドがお使いの Windows バージョンと互換性がない可能性があります。

### 変更を適用しても効果がないように見える

一部の最適化は、完全に適用されるまでに**システムの再起動**が必要です。再起動後も効果がない場合は、再度適用するか、元に戻すセクションで変更が保存されていることを確認してください。

### 元に戻すファイルがない、または壊れている

元に戻すファイルは %LocalAppData%\optimizerDuck\Revert\ に保存されています。ファイルが誤って削除されたり破損した場合は、バックアップから復元するか、事前に**システムの復元ポイント**を作成しておいてください。

### Windows Update で設定がリセットされる

Windows の機能更新プログラムによって、レジストリ値やサービス構成がデフォルトにリセットされることがあります。大型アップデート後は、アプリから以前の最適化を再適用するだけで元に戻ります。

### バグを見つけた / 機能をリクエストしたい

GitHub で [issue](https://github.com/itsfatduck/optimizerDuck/issues) を開き、お使いの Windows バージョン、適用した最適化、発生した問題をできるだけ詳しくお知らせください。機能リクエストも歓迎します。

## 技術詳細

- **フレームワーク**：.NET 10 上の WPF、WPF UI ライブラリを使用した Fluent デザイン
- **元に戻すシステム**：4 種類の元に戻すステップ（レジストリ、サービス、スケジュールタスク、シェル）、JSON 保存状態、スレッドセーフなファイル I/O
- **テーマ**：ダーク（デフォルト）、ライト、ハイコントラストモード、Mica 背景対応
- **インストーラー不要**：単一の .exe として実行、インストール不要
- **バックアップシステム**：すべての変更をローカルフォルダにバックアップ、ワンクリックで復元
- **自動検出**：最適化と機能カテゴリはリフレクション＋カスタム属性で自動検出、手動登録不要
- **テレメトリなし**：アプリはユーザーデータを一切収集しません
