## optimizerDuck이 하는 일 & 왜 중요한가

Windows는 기본적으로 잘 작동합니다. 하지만 새로 설치한 Windows에도 백그라운드 서비스, 원격 분석, 사전 설치된 앱, 예약된 작업이 포함되어 있으며, 사용하지 않더라도 계속 실행됩니다. 이들은 백그라운드에서 CPU, RAM, 디스크, 네트워크 같은 시스템 리소스를 소모합니다.

동시에 성능을 개선하고, 지연 시간을 줄이며, 더 부드러운 경험을 제공할 수 있는 많은 설정이 기본적으로 활성화되어 있지 않습니다.

optimizerDuck은 모든 것을 한곳에 모아줍니다. 레지스트리, 그룹 정책, PowerShell을 뒤질 필요 없이 Windows를 최적화하고, 불필요한 구성 요소를 제거하며, 시스템 설정을 사용자 지정하고, 일반적인 Windows 기능을 관리할 수 있습니다.

또한 내장 관리 도구를 통해 현재 실행 중인 항목을 확인하고, 필요 없는 것을 제거하며, 필요할 때마다 변경 사항을 복원할 수 있습니다.

**[Windows를 최적화해야 하는 이유는?](#하드웨어를-업그레이드하지-않고-windows를-최적화하는-이유는)**

| :--------------------------- | :---------------------------------------------------------------------------------------------------------- |
| **성능**                     | RAM 기반 서비스 호스트 튜닝, 프로세스 우선순위 조정, 키보드 지연 감소, 게임을 위한 멀티미디어 스케줄러 조정 |
| **개인정보 보호**            | Windows 원격 분석, 오류 보고, 광고 ID, 위치 추적, Cortana, Copilot, 콘텐츠 배달 제안 비활성화               |
| **GPU**                      | AMD, NVIDIA, Intel GPU용 공급업체별 레지스트리 조정, 전원 상태, 클록 게이팅, 디스플레이 지연 커버리지       |
| **전원**                     | 절전 모드 및 빠른 시작 비활성화, USB 선택적 일시 중단 해제, 고성능 전원 계획 설치, 전원 절약 비활성화       |
| **사전 설치된 앱 및 서비스** | OEM 앱 재설치 동작 차단 및 200개 이상의 Windows 서비스에 대한 시작 유형 미세 조정                           |
| **사용자 경험**              | 메뉴 표시 지연 제거, 작업표시줄 애니메이션 및 투명도 같은 시각 효과 비활성화로 더 빠른 반응성 제공          |

### 사용자 지정

레지스트리를 뒤질 필요 없이 토글, 드롭다운, 숫자 입력을 한곳에서. 네 가지 카테고리:

- **바탕화면**: 아이콘 표시 또는 숨김(내 PC, 휴지통, 네트워크, 사용자 파일, 제어판), 바로가기 화살표 오버레이 제거
- **환경 설정**: 작업표시줄 정렬, 위젯, Task View 및 End Task 버튼, 시계 초, 다크 모드, 파일 확장명, 숨김 파일, 클립보드 이력, 간단히 보기, 맞춤형 배열, 항목 확인란, 클래식 상황 메뉴 및 Bing 검색
- **게임**: 게임 모드, 게임 바, 백그라운드 녹화, 마우스 가속도, 전체 화면 최적화, 하드웨어 가속 GPU 스케줄링
- **시스템**: 부팅 시 Num Lock 활성화

### 하드웨어를 업그레이드하지 않고 Windows를 최적화하는 이유는?

성능을 개선하는 가장 빠른 방법은 항상 하드웨어를 업그레이드하는 것입니다. 더 나은 CPU, GPU, 더 많은 RAM, 더 빠른 SSD는 소프트웨어 조정만으로는 얻기 어려운 훨씬 큰 향상을 제공합니다.

하지만 하드웨어는 전체 그림의 일부일 뿐입니다.

Windows는 수억 대의 다양한 하드웨어, 작업 부하, 사용자를 위해 설계되었습니다. 그렇기 때문에 Microsoft는 최대 성능보다는 호환성, 안정성, 배터리 수명, 사용 편의성을 우선시하는 설정으로 Windows를 제공합니다.

결과적으로 기본 Windows 설치에는 많은 사용자가 필요로 하지 않을 백그라운드 서비스, 예약된 작업, 원격 분석, 절전 기능, 사전 설치된 앱이 포함되어 있습니다.

이는 Windows를 재설치한 후에 특히 두드러집니다. 모든 드라이버와 업데이트를 설치한 후에도 많은 유용한 성능 관련 설정은 그대로 남아 있고, 불필요한 백그라운드 구성 요소는 계속 실행됩니다.

Windows 최적화는 단순히 불필요한 오버헤드를 줄여 PC가 게임, 프로그래밍, 콘텐츠 제작 또는 일상 작업 등 실제로 중요한 작업에 더 많은 리소스를 사용할 수 있도록 하는 것입니다.

마법처럼 FPS가 두 배로 늘어나지는 않겠지만, 백그라운드 활동을 줄이고, 시스템 지연 시간을 낮추며, 응답성을 개선하고, 하드웨어가 더 일관되게 작동하도록 도울 수 있습니다.

이것이 많은 사람들이 더 나은 성능과 낮은 시스템 오버헤드를 위해 Linux를 추천하는 이유이기도 합니다. Windows의 소프트웨어 호환성과 익숙한 환경을 선호한다면, optimizerDuck이 운영체제를 변경하지 않고도 최상의 환경을 누릴 수 있도록 도와줍니다.

### Windows는 언제 최적화해야 하나요?

가장 좋은 시기는 **새로운 Windows를 설치한 후**입니다.

다음 순서를 권장합니다:

1. Windows를 설치합니다.
2. 모든 하드웨어 드라이버를 설치합니다(칩셋, GPU, 네트워크, 오디오 등).
3. **Windows Update**를 더 이상 업데이트가 없을 때까지 실행합니다.
4. 모든 **선택적 업데이트**를 설치합니다.
5. **Microsoft Store**를 열고 모든 내장 애플리케이션을 업데이트합니다.
6. 평소 사용하는 소프트웨어와 게임을 설치합니다.
7. 원하는 optimizerDuck 최적화를 적용합니다.
8. optimizerDuck의 내장 도구를 사용하여 불필요한 앱을 제거하고, Windows를 정리하며, 시작 프로그램을 관리합니다.

이 순서를 따르면 이후 Windows나 드라이버 업데이트가 최적화를 덮어쓰는 것을 방지할 수 있습니다.

> [!TIP]
> 많은 고급 사용자는 설정을 마친 후 Windows 자동 업데이트를 일시적으로 비활성화합니다. 주요 Windows 업데이트는 기본 설정을 복원하거나, 일부 구성 요소를 다시 설치하거나, 일부 최적화를 취소할 수 있습니다. 이렇게 할 경우 중요한 보안 업데이트를 설치하기 위해 정기적으로 Windows Update를 다시 활성화했다가 다시 비활성화하는 것을 기억하세요.

> [!NOTE]
> optimizerDuck이 제공하는 모든 최적화는 수동으로도 수행할 수 있습니다. optimizerDuck의 목표는 단순히 Windows 최적화를 더 쉽고, 안전하고, 편리하게 만드는 것입니다.

## 안전성

시스템 설정을 변경하면 위험이 따릅니다 optimizerDuck은 역변환성과 사용자 제어를 중심으로 구축되었습니다

데이터 관행에 대한 자세한 내용은 [개인정보 보호 정책](./PRIVACY.md)을 참고하세요

- **자동 백업**: 모든 변경 사항은 로컬 폴더에 되돌리기 파일을 작성합니다 개별 조정을 복원하거나 모든 것을 되돌릴 수 있습니다
- **한 클릭 되돌리기**: UI에서 한 번의 클릭으로 적용된 최적화를 취소합니다
- **위험 등급**: 각 조정은 잠재적 영향에 따라 안전, 중간, 위험으로 표시됩니다
- **기본값 미적용**: 선택할 때까지 아무것도 실행되지 않습니다 도구는 자동으로 아무것도 활성화하지 않습니다
- **복원 지점 메시지**: 첫 번째 최적화 전에 앱이 Windows 복원 지점 만들기를 제안합니다

### optimizerDuck 써도 안전한가요?

네, 안전합니다. optimizerDuck은 완전 **오픈소스**(GPL v3)라서 누구나 소스 코드를 들여다보고, 감사하고, 직접 빌드할 수 있어요. 모든 릴리즈는 **GitHub Actions**가 공개 소스에서 자동으로 빌드합니다. 숨겨진 수정 같은 건 없고, 빌드 후에 이상한 파일이 끼어들 일도 없어요. 불안하면 repo를 클론해서 `dotnet build` 한 방이면 `.exe` 뚝딱입니다.

이 앱은 **어떤** 원격 측정, 사용 데이터, 개인 정보도 수집하지 **않습니다**. [개인정보 보호 정책](./PRIVACY.md) 참고하세요.

### optimizerDuck이 진짜 성능 향상, 지연 시간 감소, 네트워크 속도 향상에 도움이 되나요?

어느 정도 도움 됩니다. optimizerDuck의 모든 최적화는 **잘 알려진 툴, 커뮤니티 가이드, 하드웨어 제조사 권장 사항을 참고한 것**이에요. AI가 지어내거나, 아무거나 막 추가한 게 아닙니다. 각 설정은 Windows가 기본적으로 너무 보수적으로 잡아놓은 실제 옵션(서비스 호스트 그룹화, GPU 전원 상태, 네트워크 제한, 프로세스 스케줄링 등)을 건드려요.

가짜 레지스트리 꼼수 같은 건 없고, 모든 변경에는 이유가 있고, 커뮤니티 테스트와 제조사 문서로 검증된 실제 효과가 있습니다.

### Windows SmartScreen / Defender가 다운로드를 막는 이유가 뭔가요?

optimizerDuck은 코드 서명이 안 돼 있어요: 코드 서명 인증서는 오픈소스 프로젝트한테는 너무 비싸거든요. Windows가 인터넷에서 받은 서명 없는 exe를 만나면 SmartScreen이 기본적으로 경고를 띄웁니다. 정상적인 현상이고, 파일이 위험하다는 뜻이 **아닙니다**.

우회하려면 **"추가 정보" > "실행"** 을 누르세요. 그래도 불안하다면:

- [소스](https://github.com/itsfatduck/optimizerDuck)에서 직접 `.exe`를 빌드해보세요
- ANY.RUN 같은 온라인 샌드박스에 파일을 올려서 확인해보세요

### 문제 생기면 되돌릴 수 있나요?

네. 모든 최적화는 적용 전에 되돌리기 파일을 만들어둡니다. UI에서 한 번 클릭으로 개별 설정을 취소하거나 전부 다 되돌릴 수 있어요. 앱에서 첫 최적화 전에 Windows 복원 지점 만들 것도 권장해줍니다.

### Windows 10이랑 Windows 11에서 되나요?

네. optimizerDuck은 **Windows 10 (x64)** 와 **Windows 11 (x64)** 를 지원합니다.

### 관리자 권한이 필요한가요?

네. 시스템 설정과 Windows 레지스트리를 건드리기 때문에 관리자 권한으로 실행해야 합니다.

### optimizerDuck이 내 데이터를 수집하나요?

아니요. 이 앱에는 원격 측정, 분석, 외부 통신 기능이 전혀 없습니다. 완전히 오프라인으로 동작하고, 데이터를 어디로도 보내지 않아요.

### 전원 계획 적용 후 작업 관리자에 CPU 100%로 표시되는 이유는? ([#29](https://github.com/itsfatduck/optimizerDuck/issues/29))

기본 전원 계획이 아닌 경우 발생하는 작업 관리자의 알려진 표시 버그, 일부 시스템에서 실제 부하와 관계없이 CPU 100%를 잘못 표시합니다. 시각적 문제일 뿐, 실제 성능에 **영향이 없으며** 과열을 유발하지 않습니다. 원하지 않으면 이 최적화를 끄기만 하면 됩니다.

## 문제 해결

> [!TIP]
> 자세한 안내와 알려진 문제 해결 방법은 [문제 해결 페이지](https://optimizerduck.vercel.app/docs/faq/troubleshooting)를 방문하세요.

### 앱이 시작되지 않거나 실행 중 충돌해요

**관리자 권한**으로 실행하고 있는지 확인하세요. optimizerDuck은 상승된 권한이 필요합니다. 계속 충돌한다면 [Releases](https://github.com/itsfatduck/optimizerDuck/releases/latest)에서 최신 버전을 다운로드하세요. 오래된 빌드는 Windows 버전과 호환되지 않을 수 있습니다.

### 변경 사항을 적용해도 효과가 없는 것 같아요

일부 최적화는 완전히 적용되려면 **시스템 재시작**이 필요합니다. 재시작 후에도 적용되지 않으면 다시 적용하거나 실행 취소 섹션에서 변경 사항이 저장되었는지 확인하세요.

### 실행 취소 파일이 없거나 손상되었어요

실행 취소 파일은 %LocalAppData%\optimizerDuck\Revert\에 저장됩니다. 파일이 실수로 삭제되거나 손상된 경우 백업에서 복원하거나 사전에 **시스템 복원 지점**을 만들어 대비하세요.

### Windows Update가 제 설정을 초기화했어요

Windows 기능 업데이트는 가끔 레지스트리 값과 서비스 구성을 기본값으로 재설정합니다. 주요 업데이트 후 앱에서 이전 최적화를 다시 적용하기만 하면 됩니다.

### 버그를 발견했어요 / 기능을 요청하고 싶어요

가능한 한 자세한 내용과 함께 GitHub에 [issue](https://github.com/itsfatduck/optimizerDuck/issues)를 열어주세요. Windows 버전, 적용한 최적화, 발생한 문제를 알려주세요. 기능 요청도 환영합니다.

## 기술 세부 사항

- **프레임워크**: .NET 10의 WPF, Fluent 디자인을 위해 WPF UI 라이브러리 사용
- **되돌리기 시스템**: 4가지 되돌리기 단계 유형(레지스트리, 서비스, 예약된 작업, 셸)과 JSON 지속 상태 및 스레드 안전 파일 I/O
- **테마**: 다크(기본값), 라이트, 고대비 모드 및 Mica 백드롭 지원
- **설치 프로그램 없음**: 단일 .exe로 실행되며 설치가 필요 없습니다
- **백업 시스템**: 모든 변경에 대한 폴더 기반 백업이 한 번의 클릭으로 복원됩니다
- **검색**: 최적화 및 기능 범주는 리플렉션 + 사용자 지정 속성을 통해 자동으로 검색되며 수동으로 등록할 필요가 없습니다
- **원격 분석 없음**: 앱은 사용자 데이터를 수집하지 않습니다
